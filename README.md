# AI知识学习记忆系统（ai-quiz-system）

基于艾宾浩斯遗忘曲线的**AI知识测验与复习管理系统**，通过飞书多维表格做数据库，自动出题、批改、追踪学习进度，帮你用最少的复习次数形成长期记忆。

## 一、项目简介

### 这个系统做了什么？

你把学习内容告诉AI，系统自动：
1. **提炼知识点**到飞书知识库（多维表格）
2. 按**艾宾浩斯遗忘曲线**安排复习节奏（添加后第1/3/7/15/30天）
3. **自动出题**：选择/填空/简答混合，薄弱的重点考、已掌握的少考
4. **自动批改**：答对推进复习轮次，答错归零重新走曲线并标记为薄弱点
5. **追踪进度**：随时查看总知识数、掌握状态、薄弱点情况
6. **自我迭代**：每次项目修改自动提取经验，失败教训和成功做法都能沉淀复用

### 适用场景

- 系统性学习AI/编程/任何知识
- 备考学习，需要长期记忆的知识体系
- 碎片化学习，需要定期复习巩固

## 二、核心功能

| 功能 | 说明 | 触发关键词 |
|------|------|-----------|
| **出题测验** | 从飞书知识库按5级优先级筛选今日待考知识点，自动生成试卷 | "开始测试""来一套题""考我""每日测验""出题" |
| **批改写回** | 逐题批改后，按艾宾浩斯规则计算更新，自动写回飞书 | 用户答完题后自动触发 |
| **知识点入库** | 将新学习内容提炼为知识点，写入飞书知识库 | "今天学了""添加知识点""新内容" |
| **进度查询** | 查看总知识点数、掌握状态分布、近期到期题数等 | "查看进度""学习情况" |
| **项目修改迭代** | 修改项目规则/逻辑/配置时，按标准协议执行+自动提取经验 | "修复""修改""检查逻辑""核查""优化" |

### 出题优先级（5级筛选，到期优先，未到期仅补位）

| 优先级 | 来源标签 | 说明 | 是否标🔥 | 推进轮次 |
|--------|---------|------|---------|---------|
| P1 | 薄弱点到期 | 答错过的知识点，已到复习日期（最高优先级） | ✅ | ✅ 答对+1 |
| P2 | 到期常规 | 没答错过的知识点，已到复习日期 | ❌ | ✅ 答对+1 |
| P3 | 新知识点 | 昨天刚入库，今天第一次考 | ❌ | ✅ 答对+1 |
| P4 | 未到期巩固 | 答错过的知识点，提前巩固（**仅当题量不足时才出题**） | ✅ | ❌ 不推进 |
| P5 | 随机巩固 | 题量仍不足时从已学知识点随机补充 | ❌ | ❌ 不推进 |

### 知识点分级

| 级别 | 含义 | 出题策略 |
|------|------|---------|
| ★★★ 核心 | 必掌握的核心知识 | 优先出简答题，高频考察 |
| ★★☆ 重要 | 应该掌握的重点知识 | 出选择/填空，正常考察 |
| ★☆☆ 补充 | 了解即可的补充知识 | 只出选择题，不主动出题 |

## 五、工作流程

系统共有5个工作流，AI会根据你的输入自动判断走哪个流程：

### 流程A：出题测验（你说"开始测试""考我"等）

```
你触发关键词 → AI执行quiz_pull.py → 从飞书筛选今日题目 →
读取风险提示+经验预警 → 按格式规范出题 → 12项自检 → 展示试卷 → 你开始答题
```

🔴 出题必须**严格按知识点清单顺序**，不得打乱顺序或按题型分组，否则批改时会更新错知识点。

🔴 出题前必须读取 `risk-alerts.md`（预警已知风险）和 `quiz-experience.md`（参考核心规则）。

### 流程B：批改写回（你答完所有题目后）

```
读取批改规则+经验预警 → AI逐题批改 → 写入grading.json → 执行quiz_push.py →
自动计算艾宾浩斯A/B/C三类更新 → 批量写回飞书 → 批改自检 → 展示批改总结
```

更新规则（A/B/C三种类型）：

| 类型 | 条件 | 结果 |
|------|------|------|
| **类型A** | 薄弱点到期/到期常规/新知识点 + 答对 | 轮次+1，状态推进（轮次≥5→已掌握） |
| **类型B** | 未到期巩固/随机巩固 + 答对 | 轮次不变（不推进），仅更新正确次数 |
| **类型C** | 任何情况 + 答错 | 轮次归零，标记薄弱点，明天重新考 |

🔴 批改前必须读取 `risk-alerts.md`（预警已知风险）、`quiz-experience.md`（参考核心规则）、`grading-checklist.md`（11项自检）。

### 流程C：知识点入库（你说"今天学了XXX"）

```
读取入库规则+经验预警 → AI审查科学性 → 查重去重 → 结构化归属（L0~L4分类） →
重要性分级（★★★/★★☆/★☆☆） → 写入飞书 → 反馈入库结果
```

🔴 入库前必须读取 `risk-alerts.md`（预警已知风险）和 `quiz-experience.md`（参考核心规则）。

### 流程D：进度查询（你说"查看进度"）

```
读取飞书知识库 → 统计：总知识数、掌握状态分布、分级分布、薄弱点数量 →
用自然语言总结并给出建议
```

### 流程E：Skill修改与迭代（你说"修复""修改""核查""优化"等）

```
判断变更等级(P0-P3) → 搜索关联位置 → 查阅映射表 → 自指检查 →
执行修改 → 一致性校验 → 联动检查 → CHANGELOG → 二次校验 → 经验提取与沉淀
```

> 流程E是项目自我迭代的入口，确保每次修改都有据可查、经验可复用。详细步骤见 `SKILL.md` 第七章。

## 六、使用方法

### 前置条件

1. Python 3.x 已安装
2. lark-cli 已安装并完成飞书授权（路径见 `scripts/config.py`）
3. 飞书多维表格（知识库）已创建，BASE token 和 TABLE id 已配置到 config.py

### 日常使用

**开始学习（出题）：**
```
对AI说："开始测试" 或 "来一套题"
```

**答题：**
按顺序在同一个回复中写答案即可。

**添加新知识：**
```
对AI说："今天学了XXX" + 附上学习材料（文本/截图/链接）
```

**查看进度：**
```
对AI说："查看进度"
```

**修改项目规则或修复问题：**
```
对AI说："修复 XXX 问题" 或 "核查 XXX 逻辑" 或 "优化 XXX"
```

### 脚本命令速查

| 命令 | 作用 | 执行时机 |
|------|------|---------|
| `python scripts/quiz_pull.py` | 从飞书拉取知识点，生成今日题目 | 出题前（流程A第一步） |
| `python scripts/quiz_push.py` | 批改后计算更新，写回飞书 | 批改后（流程B第四步） |
| `python scripts/quiz_push.py --dry-run` | 测试模式，只计算不写飞书 | 调试验证 |
| `python check_consistency.py` | 全量一致性检查（79+项） | 修改任何文件后 |
| `python check_consistency.py where 关键词` | 搜索关键词在所有文件中的位置 | 修改前定位所有关联引用 |
| `python check_consistency.py linkage` | 联动检查：发现文件间读写断裂 | 新增文件/新增经验后 |

## 七、三位一体防护体系

为防止修改时遗漏相关联位置，项目设计了**轻量级三位一体防护机制**：

| 防护层 | 文件 | 作用 | 类比 |
|--------|------|------|------|
| **第一层（源头）** | `scripts/config.py` | 所有常量（路径/标签/阈值/字段名/格式串）的唯一来源，改一处全生效 | 中央数据库 |
| **第二层（地图）** | `content-map.md` | 人工查阅的关联地图，标注每个内容的主定义位置和所有引用位置（覆盖全部21个文件） | 导航地图 |
| **第三层（兜底）** | `check_consistency.py` | 自动校验79+项规则，拦截遗漏和不一致；附带 `where` 关键词搜索和 `linkage` 联动检查 | 安检系统 |

### 修改规则（必读）

| 阶段 | 操作 | 命令 |
|------|------|------|
| 修改前 | 搜索所有关联位置 | `python check_consistency.py where 关键词` |
| 修改前 | 查阅映射表 | 读取 `content-map.md` |
| 修改后 | 一致性校验 | `python check_consistency.py`（0错误才算完成） |
| 新增文件/经验后 | 联动检查 | `python check_consistency.py linkage` |

> 详细修改协议（E0-E8完整步骤）请参阅 `SKILL.md` 第七章"流程E"和第十章"Skill修改协议"。

## 八、经验沉淀与自我迭代

### 三级经验沉淀体系

每次项目修改（流程E）后，系统自动从修改中提取经验，分流到三个经验库：

| 层级 | 文件 | 定位 | 写入时机 |
|------|------|------|---------|
| 核心规则层 | `references/quiz-experience.md` | 测验专用规则，与铁律同等效力 | 发现新的核心规则时 |
| 最佳实践层 | `references/best-practices.md` | 成功修复的有效做法 | 修改成功后提取 |
| 风险提示层 | `references/risk-alerts.md` | 已知踩坑点+规避方法 | 修改失败/引入新问题后提取 |

### 读写闭环

经验文件不是"写了就完"——出题（A2）、批改（B1）、入库（C1）的每一步都会**先读取经验文件**，确保已知经验和风险在执行前被参考。

### 不满意更新机制

如果用户对修改结果不满意（无论立即反馈还是一段时间后在使用中发现问题），重新修改后必须更新已沉淀的经验：
- 完全推翻 → 原经验标注 `❌已废弃`
- 部分修正 → 补充到"注意事项"中
- 新经验标注 `[观察期]`，3次验证后改为 `✅已固化`

## 九、文件联动检查

项目内置 **联动检查工具**，专门解决"新增文件/经验后不知道还需要关联到哪里"的问题：

```
python check_consistency.py linkage
```

自动检查：
- 14个关键文件的**读写闭环**（谁写入谁读取）
- 是否有**新增文件未纳入联动监控**
- 是否有文件**只被写入但从未被任何流程读取**

示例输出：
```
✅ references/quiz-experience.md — 读写闭环正常
⚠️ references/新文件.md — 可能存在读写断裂！未找到读取方引用
```

## 十一、重要注意事项

### ⚠️ 绝对不能做的事（红线）

1. **🔴打乱题目顺序**：出题时必须按知识点清单顺序，第N题必须对应清单[N]，否则批改会更新错知识点
2. **省略填空下划线**：填空题空白处必须是 `____（N）____` 格式，前后都要有4条下划线
3. **跳过脚本**：出题前必须跑 quiz_pull.py，批改后必须跑 quiz_push.py
4. **编造知识点**：所有题目必须来自 today_quiz.json 中的知识点核心内容
5. **修改 grading.json 格式**：必须严格按照模板格式
6. **硬编码常量**：所有可配置值写在 config.py 中，脚本通过 import 引用
7. **修改后跳过经验提取**：每次执行流程E后必须走E8经验提取，不得跳过
8. **新增文件不跑联动检查**：新增文件或经验后必须运行 `python check_consistency.py linkage`

### ⚠️ 常见问题

**Q：批改时题号对不上知识点怎么办？**
A：这是出题时打乱了知识点顺序导致的问题。现在出题指令最前面有🔴强制顺序要求，自检清单有第0条顺序检查，quiz_push.py会在批改前打印"题号→知识点ID对应表"供核对。如果出现题号不对应，说明出题步骤未遵守顺序规则。

**Q：填空题下划线为什么格式这么严格？**
A：出题时如果填空题格式不对（如只写（1）不写下划线），会影响自检清单的自动检查，也可能导致批改时的解析问题。这是实际使用中最高频的错误，检查清单第1条专门针对这个问题。

**Q：修改配置后怎么知道有没有遗漏？**
A：运行 `python check_consistency.py`，会自动检查79+项规则，包括文件存在性、常量完整性、路径正确性、标签一致性、格式规范、脚本引用、联动闭环等，有错误会明确指出。

**Q：新增了文件或经验条目，怎么确保关联正确？**
A：运行 `python check_consistency.py linkage`，会自动检查14个关键文件的读写闭环，发现"被写入但从未被读取"的断裂会明确提示并给出修复建议。
