# 新知识点入库规则

> 当用户发来新学习内容时，必须严格按照本规则处理。
> ⚠️ 科学性审查优先于查重，查重优先于入库。

---

## 术语前置解释（零基础必读）

| 术语 | 解释 |
|------|------|
| 知识点（KP） | Knowledge Point，飞书多维表格中一条独立的知识卡片，有唯一ID，是艾宾浩斯记忆的最小单元 |
| 入库 | 将新学习的知识点按照规范格式写入飞书多维表格的过程 |
| 查重 | 检查新内容是否与知识库中已有知识点重复，避免重复创建 |
| lark-cli | 飞书命令行工具（Lark Command Line Interface），用于通过脚本读写飞书文档和多维表格 |
| 多维表格 | 飞书的一种在线数据表格，类似数据库，用于存储结构化的知识点数据 |
| 正交分类 | 正交（Orthogonal）指"互相独立、不重叠"。正交分类即从多个互不重叠的维度给知识点打标签，弥补层级分类的不足 |
| ID | Identifier的缩写，即唯一标识符，每个知识点都有一个6段式的唯一ID |

## 一、入库7步流程概览

```
用户提交学习材料
    │
    ▼
步骤1：科学性/时效性审查 ── 发现错误/过时/翻译问题 → 标注并纠正
    │
    ▼
步骤2：查重去重 ── 逐条比对现有知识库，判断4种重叠情况
    │
    ▼
步骤3：结构化归属判断 ── 判断属于哪个章节/小节，还是新板块
    │
    ▼
步骤4：重要性分级 ── 三问判定法确定★★★/★★☆/★☆☆
    │
    ▼
步骤5：知识点提炼 ── 分配ID、编写核心内容、确定标签
    │
    ▼
步骤6：写入飞书 ── 通过lark-cli写入多维表格
    │
    ▼
步骤7：反馈 ── 按模板输出入库反馈
```

⚠️ 步骤顺序不能调换，必须先做科学性审查，再查重，再入库。

## 二、步骤1：科学性/时效性审查

每个候选知识点入库前**必须先通过6项审查**，这是所有后续步骤的前提。

| 检查项 | 含义 | 处理动作 |
|--------|------|---------|
| 事实错误 | 表述与公认教材/论文/官方文档矛盾 | 标注原文错误位置 + 给出正确内容 |
| 过时信息 | 被后续研究推翻的观点、已废弃的技术/工具名 | 标注"已过时" + 给出当前主流状态 |
| 翻译偏差 | 中文术语与英文原文含义不对应 | 给出英文原文 + 推荐准确翻译 |
| 以偏概全 | 把某个特例当成普遍规律 | 补充适用范围说明，"这只在XXX条件下成立" |
| 笔误/数字错误 | 公式写错、年份/百分比数字错误 | 直接修正 |
| 个人观点冒充事实 | 作者主观判断未声明为观点 | 标注"这是作者的个人观点" |

全部通过后再进入下一步。

## 三、步骤2：查重去重4种策略

### 3.1 比对方法
1. 标题关键词匹配：在现有KP的"知识点标题"和"核心内容"中搜索核心概念词
2. 章节/小节比对：同小节是高重叠信号
3. 语义判断："如果用户已经掌握了现有KP，是否等于已经掌握了这个新内容"

### 3.2 四种处理策略

| 重叠程度 | 判定标准 | 处理方式 |
|---------|---------|---------|
| **完全重复** | 同一个知识点，没有比现有KP更多信息 | **跳过不创建**。反馈中告知用户已被哪个ID覆盖 |
| **重合但有补充价值** | 讲同一件事，但有新例子、新角度、更清晰表述 | **丰富现有KP**：把有价值的部分追加到现有KP核心内容末尾，不新增KP |
| **部分交叉但角度独立** | 有交集但不完全重合，可独立成一个值得单独记忆的知识点 | **新增独立KP**：标题标注与已有KP的关系 |
| **完全无重叠** | 知识库中完全未涉及的新概念 | **正常入库** |

### 3.3 实操示例

**场景A——完全重复：**
> 用户发来："机器学习就是让计算机从数据中学习，而不是人手工写规则"
> → 命中ML-001 → 跳过不新建

**场景B——重合但有补充：**
> 用户发来："过拟合不光出现在机器学习，在统计学里也有这个概念，可以追溯到高斯"
> → 命中ML-005，补充统计学历史视角 → 追加到现有KP内容

**场景C——交叉但独立：**
> 用户发来："k近邻算法在高维数据中会出现'维数灾难'"
> → 命中ML-002（k近邻基础），但这是进阶局限性 → 新建独立KP

**场景D——完全无重叠：**
> 用户发来："集成学习是一种用多个模型组合提升准确率的方法"
> → 未命中 → 正常入库

### 3.4 用户纠错机制
如果用户认为去重判定有误（"我觉得XX不应该被跳过，它和已有KP讲的不一样"）：
1. 重新评估做第二轮语义判断
2. **以用户判断为准**，如果用户坚持独立则新建KP入库

## 四、步骤3：结构化归属判断

### 4.1 归属判断三选一
1. **纳入现有小节**：内容属于现有某章某小节 → 直接填对应字段
2. **现有章内新增小节**：属于某章但不属于现有任何小节 → 先在"小节分组"字段追加新选项，再写入
3. **开启新章节**：不属于现有任何一章 → 先在"章节"字段追加新选项，同时创建至少2个初始小节，再写入

⚠️ **禁止把新知识点随意扔到"其他"兜底分类**——正常情况下必须判断归属到现有章节或开新章节。

### 4.2 当前6级分类体系
- **L0阶段**：共10个阶段，当前是"阶段1：AI通识认知"
- **L1领域**：统一为"人工智能"
- **L2模块**：AI导论(INTRO)、搜索博弈(SEARCH)、机器学习(ML)、AI伦理(ETHIC)
- **L3主题**：如机器学习下的概率统计基础(PROB)、基础概念(CORE)、经典模型(MODEL)、深度学习(DL)
- **L4章节**：如深度学习下的神经网络基础(NN)、计算机视觉(CV)、序列模型/自然语言处理(SEQ/NLP)、生成模型(GEN)、大语言模型(LLM)
- **知识标签**：多选字段，用于跨分类正交标签（如"Transformer""生成模型"）

分类细节参考：`d:\AI\AI学习内容记忆测试\ai-quiz-system\references\04-knowledge-taxonomy.md`

## 五、步骤4：重要性三级分级（三问判定法）

对每个候选知识点，依次回答三个问题：

| 问题 | 是→倾向 | 否/一般→倾向 |
|------|---------|-------------|
| **第一问：如果没学这个知识点，会影响理解后续更核心的AI内容吗？** | ★★★ | 不影响但有助认知→★★☆；末端信息→★☆☆ |
| **第二问：讨论AI时，这个概念被提到的频率有多高？** | 几乎所有入门材料都提→★★★ | 特定领域涉及→★★☆；仅本材料→★☆☆ |
| **第三问：这个概念是"稳定的知识"还是"会过时的信息"？** | 30年前就有、30年后还在→★★★ | 当前主流但可能被替代→★★☆；产品名/年份/具体数字→★☆☆ |

### 定级规则
- 两问倾向★★★且无一问倾向★☆☆ → ★★★核心
- 两问倾向★☆☆且无一问倾向★★★ → ★☆☆补充
- 其他情况 → ★★☆重要

### 定级示例
| 知识点 | 一问 | 二问 | 三问 | 最终 |
|--------|-----|-----|-----|------|
| 线性回归 | 是 | 极高 | 200年稳定 | ★★★ |
| 具体换脸工具名 | 否 | 极低 | 产品名 | ★☆☆ |
| 零频率问题 | 否 | 中 | 数学稳定 | ★★☆ |
| 过拟合 | 是 | 极高 | 长期稳定 | ★★★ |

### 各级出题策略
- **★★★核心**：适当多出简答题促进深度思考，到期必出，随机巩固优先抽
- **★★☆重要**：到期才出题，题量有空间时纳入巩固，可出简答题
- **★☆☆补充**：只出选择题，不主动出题，无题可出时才考虑

## 六、步骤5-6：知识点提炼与写入飞书

### 6.1 ID命名规则
格式：`{L0阶段号}-{L1缩写}-{L2缩写}-{L3缩写}-{L4缩写}-{两位序号}`
- 阶段1所有内容L1为AI
- 缩写用大写英文/拼音缩写，保持简洁
- 序号从01开始递增

**示例：**
- `1-AI-ML-CORE-OVERFIT-01` = 阶段1→人工智能→机器学习→基础概念（CORE）→过拟合→第1个
- `1-AI-ML-DL-LLM-01` = 阶段1→人工智能→机器学习→深度学习→大语言模型→第1个

> **概念包含关系说明**：ID保持6段格式不变（LLM直接挂在DL下作为L4章节），但概念上LLM ⊂ 基础模型 ⊂ 生成模型 ⊂ 深度学习（⊂ 表示"属于"）。这种多层归属需要通过**L4章节字段**和**知识标签**来体现，例如给LLM知识点打上`生成模型`、`基础模型`、`LLM`标签。

### 6.2 字段初始值
| 字段 | 初始值 |
|------|-------|
| 添加日期 | 当天日期（YYYY-MM-DD） |
| 掌握状态 | "⚪未测试" |
| 复习轮次 | 0 |
| 下次复习日期 | 添加日期+1天（首次复习次日） |
| 正确次数/错误次数 | 0 |
| 错题本/薄弱点 | false |
| 薄弱点描述 | null |

### 6.3 核心内容编写要求
- ≤200字，通俗语言
- 先给定义，再给核心要点，最后给关键例子/误区
- 专业名词首次出现附解释
- 可用全角特殊字符｜＜＞避免命令行解析问题

## 七、步骤7：入库反馈格式

全部处理完成后，严格按照`d:\AI\AI学习内容记忆测试\ai-quiz-system\assets\templates\ingest-feedback.md`格式输出反馈，必须包含：
1. 科学性/时效性审查结果
2. 查重去重结果（每个候选点的判定）
3. 结构化归属（每个点放在了哪个章节）
4. 重要性分级（按★★★/★★☆/★☆☆分组列出）
5. 入库统计数字
6. 一句学习建议
