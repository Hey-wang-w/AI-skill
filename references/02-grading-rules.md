## 一、艾宾浩斯常数与基础规则

> **术语解释**：**艾宾浩斯遗忘曲线**是德国心理学家艾宾浩斯发现的记忆规律——学完新知识后遗忘速度先快后慢，按照"第1天、第3天、第7天、第15天、第30天"的间隔反复复习，就能把短期记忆转化为长期记忆。本系统用这个规律自动计算下次复习日期。

### 1.1 间隔数组
```python
CUMULATIVE = [1, 3, 7, 15, 30]  # 单位：天，索引0对应轮次0→1间隔1天
```

| 当前轮次（答对前） | 答对后下次复习间隔 | 新轮次 |
|:--:|:--:|:--:|
| 0（首测/答错后） | 1天后 | 1 |
| 1 | 3天后 | 2 |
| 2 | 7天后 | 3 |
| 3 | 15天后 | 4 |
| 4 | 30天后 | 5（🟢已掌握） |
| 任何轮次答错 | 1天后 | 0（🔴待复习） |

### 1.2 批改前准备
1. 确认`d:\AI\AI学习内容记忆测试\ai-quiz-system\today_quiz.json`存在（出题时生成的）
2. 逐题判断对错，给出错因分析
3. 严格按照`d:\AI\AI学习内容记忆测试\ai-quiz-system\assets\templates\grading-result.json`格式生成结果文件
4. 执行`d:\AI\AI学习内容记忆测试\ai-quiz-system\scripts\quiz_push.py`脚本写回飞书，**不得手动计算日期和轮次**

## 二、类型A：薄弱点到期 / 到期常规 / 新知识点 → 答对

### 触发条件
来源标签为`[薄弱点到期]`、`[到期常规]`或`[新知识点]`的题目，用户答对。

> **术语解释**：
> - **薄弱点到期**：已到艾宾浩斯复习日期，且该知识点之前答错过（标记为薄弱点=true）。
> - **到期常规**：已到艾宾浩斯复习日期，但该知识点不是薄弱点（正常复习轮次推进）。
> - **新知识点**：刚入库、第一次被测试的知识点。

### 字段更新规则（脚本自动处理，AI无需手动计算）

| 字段 | 更新方式 |
|------|---------|
| 复习轮次 | +1 |
| 正确次数 | +1 |
| 掌握状态 | 轮次≥5 → "🟢已掌握"；否则 → "🟡学习中" |
| 下次复习日期 | 按FIX#1防回溯公式计算（见第五章） |
| 错题本 | 不变 |
| 薄弱点（原薄弱点） | 追加"正式测验YYYY-MM-DD到期答对"；从最后一次`[重置]`后统计连续到期答对次数；≥2次 → 薄弱点=false，清除描述 |
| 薄弱点（原非薄弱点） | 不变 |

## 三、类型B：未到期巩固 / 随机巩固 → 答对

### 触发条件
来源标签为`[未到期巩固]`或`[随机巩固]`的题目，用户答对。

> **术语解释**：
> - **未到期巩固**：知识点尚未到艾宾浩斯复习日期，但因当日题量不足被提前抽中巩固。
> - **随机巩固**：从已学知识点中随机抽取用于巩固练习，答对不推进轮次（FIX#4保护）。

### 字段更新规则（脚本自动处理）

| 字段 | 更新方式 |
|------|---------|
| 复习轮次 | **不变（FIX#4保护：不推进轮次！）** |
| 正确次数 | +1 |
| 下次复习日期 | **不变** |
| 掌握状态 | 若原为"⚪未测试"→ "🟡学习中"；否则不变 |
| 错题本 | 不变 |
| 薄弱点（未到期巩固） | 追加"正式测验YYYY-MM-DD未到期巩固答对"；**不计入连续答对清除，薄弱点标记保持true（FIX#2）** |

## 四、类型C：任何情况 → 答错

### 触发条件
任何来源标签的题目，用户答错：
- 选择题选错
- 填空题核心术语错误
- 简答题缺要点超过一半

### 字段更新规则（脚本自动处理）

| 字段 | 更新方式 |
|------|---------|
| 复习轮次 | = 0（直接打回第0轮，重新走1→3→7→15→30） |
| 错误次数 | +1 |
| 错题本 | = true |
| 掌握状态 | = "🔴待复习" |
| 下次复习日期 | = 明天（今天+1天） |
| 薄弱点 | = true（若还没标则打上） |
| 薄弱点描述 | 追加："正式测验YYYY-MM-DD答错 [重置]：{一句话错因分析}"<br>⚠️ `[重置]`标记表示之前的答对记录全部作废，连续答对计数从这里重新开始（FIX#3） |

### 错因分析要求
错题的错因必须写清楚：
- 用户选了什么/答了什么
- 正确答案是什么
- 核心错误点是什么（概念混淆？记忆模糊？理解偏差？）
- 必要时补充1-2句讲解

**示例：**
> 错因：误选D"用测试集反复调参"——测试集只能在最后评估用一次，反复调参会导致"偷看答案"式过拟合。正确答案是B。

## 五、FIX#1~#7保护规则详解

> 这7条是历史上修复的逻辑Bug，必须严格遵守。

| 编号 | 保护的问题 | 核心规则 |
|------|----------|---------|
| **FIX#1** | 答错后轮次归零，按添加日计算的日期早于今天，导致回溯加速催熟 | 计算公式：`date_by_add = 添加日期 + CUMULATIVE[新轮次]`（取下一轮间隔天数）<br>若 `date_by_add <= 今天` → 改用 `今天 + CUMULATIVE[新轮次]`<br>保证：不催熟、归零后首次复习在明天 |
| **FIX#2** | 薄弱点"未到期巩固答对"被错误计入连续答对，导致薄弱点提前解除 | **只统计"到期答对"**，未到期巩固答对只记录，不计入连续清除计数 |
| **FIX#3** | 答错后原有的连续答对计数没清零，导致薄弱点解除逻辑错误 | 答错必须追加`[重置]`标记，连续计数从标记后重新算起 |
| **FIX#4** | 随机巩固抽到的未到期题答对后错误推进了轮次 | 随机巩固答对**不推进轮次、下次日期不变** |
| **FIX#5** | 同一知识点在多个优先级层被重复选中，同一题出多次 | 每层选完标记已选ID，下层排除已选ID |
| **FIX#6** | 候选超过20题时裁题顺序不明确 | 按优先级P1>P2>P3>P4>P5截断：P1（薄弱点到期）优先保留，P4/P5（未到期）最先裁，★☆☆优先裁 |
| **FIX#7** | FIX#1公式下标用错（用了`CUMULATIVE[新轮次-1]`即刚完成轮次的间隔，导致下次复习日期=今天，次日即到期天天催熟） | 修正为`CUMULATIVE[新轮次]`（取下一轮间隔）；同时`< 今天`改为`<= 今天`防止边缘情况 |
