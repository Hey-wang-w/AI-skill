# AI知识学习记忆系统 Skill

> 本Skill是管理AI知识学习、出题测验、批改复习、知识点入库的唯一执行依据。
> ⚠️ 所有规则必须严格遵守，不得自行简化或跳过步骤。

## 一、术语前置解释（零基础必读）

| 术语 | 解释 |
|------|------|
| 知识点（KP） | Knowledge Point，飞书表格里一条独立的知识卡片，有唯一ID |
| 艾宾浩斯遗忘曲线 | 记忆规律：学完后遗忘先快后慢，按"添加后第1/3/7/15/30天"累计间隔复习形成长期记忆；答错则归零重走（脚本自动计算，AI无需手动算日期） |
| 薄弱点 | 答错的知识点，会被高频考察直到连续2次到期答对 |
| 薄弱点到期 | 来源标签之一：该知识点已到艾宾浩斯复习日期，且该知识点被标记为薄弱点（之前答错过），需标🔥 |
| 到期常规 | 来源标签之一：该知识点已到艾宾浩斯复习日期，但不是薄弱点（正常复习），不标🔥 |
| 未到期巩固 | 来源标签之一：该知识点尚未到期，但因题量不足被提前抽中巩固，若该知识点是薄弱点则标🔥 |
| 新知识点 | 来源标签之一：刚入库、第一次被考到的知识点，不标🔥 |
| 随机巩固 | 来源标签之一：从已学过的知识点中随机抽取巩固，不推进轮次（FIX#4保护），不标🔥 |
| 正式测验 | 更新飞书复习记录的测验；纯练习不改动飞书 |
| ★★★/★★☆/★☆☆ | 知识点重要性三级：核心/重要/补充，决定出题频率 |
| FIX#1~#6 | 6个历史逻辑Bug修复的保护规则，脚本中已实现，批改时需注意理解规则即可 |

## 三、流程A：出题测验（高频核心流程）

### 执行步骤（必须严格按顺序执行，不得跳步）

**步骤A1：执行拉题脚本**
- ⚠️ 强制运行脚本，不得跳过：
  ```powershell
  python d:\AI\AI学习内容记忆测试\ai-quiz-system\scripts\quiz_pull.py
  ```
- 脚本会自动从飞书拉取知识点、按优先级筛选、生成`today_quiz.json`和出题指令。
- 等待脚本执行完成，确认输出"出题指令已保存"。

**步骤A2：读取格式规范**
- ⚠️ 必须读取：`d:\AI\AI学习内容记忆测试\ai-quiz-system\references\01-question-format.md`
- 重点关注：填空题的下划线格式要求、选择题选项规则、专业名词解释要求。

**步骤A3：读取试卷模板**
- ⚠️ 必须读取：`d:\AI\AI学习内容记忆测试\ai-quiz-system\assets\templates\quiz-paper.md`
- 严格按照模板格式输出，不得自行创造格式。

**步骤A4：读取今日题目数据**
- 读取脚本生成的：`d:\AI\AI学习内容记忆测试\ai-quiz-system\today_quiz.json`
- 读取脚本生成的：`d:\AI\AI学习内容记忆测试\ai-quiz-system\出题指令.txt`（里面有完整的知识点核心内容）

**步骤A5：出题（🔴最高优先级：不得打乱知识点顺序！）**
- 🔴**必须严格按照出题指令中"今日待出题知识点清单"的顺序出题，不得打乱顺序、不得按题型分组、不得调整题目顺序**
- 🔴**第N题必须对应知识点清单[N]，题目编号N和知识点ID必须与清单完全一致**（违反此条会导致批改时对错更新到错误知识点！）
- 题型混合在原顺序中自然分布即可，选择题正确选项位置可以随机但题目顺序不变
- 按照知识点核心内容出题，混合选择/填空/简答题型
- 题型比例参考出题指令中的建议
- ★☆☆知识点只出选择题，不出填空/简答
- 来源标签五选一：`[薄弱点到期]`、`[未到期巩固]`、`[到期常规]`、`[新知识点]`、`[随机巩固]`
  - **薄弱点题**（薄弱点=true，即来源为`[薄弱点到期]`或`[未到期巩固]`）末尾标🔥
  - **到期常规题**（来源为`[到期常规]`）不标🔥（这类是正常到期复习，不是薄弱点）
  - `[新知识点]`和`[随机巩固]`也不标🔥

**步骤A6：强制自检卡点（⚠️ 不通过不能输出）**
- ⚠️ 必须逐项核对：`d:\AI\AI学习内容记忆测试\ai-quiz-system\assets\checklists\question-checklist.md`
- 11项全部检查通过后，才能输出最终试卷给用户。
- 特别注意第1项：填空题必须有`____（N）____`格式的下划线+编号！

### 输出要求
- 开头输出试卷头（题型/来源/分级统计），来源分布必须区分薄弱点和常规到期：`🔥薄弱点题 X 题（薄弱点到期 A + 未到期巩固 B）+ 📅到期常规 C 题 + ⚪新知识点 D 题 + 🎲随机巩固 E 题`
- 题与题之间空一行
- 专业名词首次出现必须附括号中文解释
- 最后告诉用户："请按顺序作答，可以一次性写在一个回复里"

## 四、流程B：批改写回（用户答完题后）

### 执行步骤

**步骤B1：读取批改规则**
- ⚠️ 必须读取：`d:\AI\AI学习内容记忆测试\ai-quiz-system\references\02-grading-rules.md`
- 重点关注：FIX#1~#6保护规则、A/B/C三种类型的更新方式、薄弱点状态机。

**步骤B2：逐题批改**
- 判断每道题对错：
  - 选择题：选对即对，选错即错
  - 填空题：核心术语意思答对即给分，不要求一字不差
  - 简答题：命中要点即给分，缺要点超过一半算错
- 错题必须写清楚：正确答案是什么、用户错在哪里、原因分析。

**步骤B3：生成grading.json**
- ⚠️ 必须严格按照模板格式：`d:\AI\AI学习内容记忆测试\ai-quiz-system\assets\templates\grading-result.json`
- 保存位置：`d:\AI\AI学习内容记忆测试\ai-quiz-system\grading.json`
- 格式：`[{"题号": N, "正确": true/false, "错因": "错误原因分析（错题必填）"}]`

**步骤B4：执行写回脚本**
- ⚠️ 强制运行脚本，不得跳过：
  ```powershell
  python d:\AI\AI学习内容记忆测试\ai-quiz-system\scripts\quiz_push.py
  ```
- 脚本自动完成：轮次更新、下次复习日期计算、薄弱点状态、写回飞书。

**步骤B5：输出批改总结**
- 按照`d:\AI\AI学习内容记忆测试\ai-quiz-system\assets\templates\grading-summary.md`格式输出
- 必须包含：正确率、薄弱点变化、今日错题、下次预告

## 五、流程C：新知识点入库（用户发学习内容时）

### 执行步骤

**步骤C1：读取入库规则**
- ⚠️ 必须读取：`d:\AI\AI学习内容记忆测试\ai-quiz-system\references\03-knowledge-ingest.md`
- 重点关注：科学性审查6项标准、查重4种策略、三问定级法。

**步骤C2：读取分类体系**
- ⚠️ 必须读取：`d:\AI\AI学习内容记忆测试\ai-quiz-system\references\04-knowledge-taxonomy.md`
- 确定知识点归属的L0-L4层级、分配正确的ID格式。

**步骤C3：执行7步入库流程**
1. 科学性/时效性审查：检查事实错误、过时信息、翻译偏差
2. 查重去重：比对现有知识库，判断是完全重复/重合补充/交叉独立/全新
3. 结构化归属：判断属于哪个章节，还是需要新建小节/新章
4. 重要性分级：用三问判定法确定★★★/★★☆/★☆☆
5. 知识点提炼：分配ID、编写核心内容、确定标签
6. 写入飞书：通过lark-cli写入多维表格
7. 输出反馈：按照模板反馈入库结果

**步骤C4：入库反馈**
- ⚠️ 严格按照：`d:\AI\AI学习内容记忆测试\ai-quiz-system\assets\templates\ingest-feedback.md`格式输出
- 必须包含：科学性审查结果、查重结果、归属结构、分级、入库统计

## 八、硬性规则红线（违反即严重错误）

⚠️ 以下规则绝对不能违反：

1. **🔴不得打乱题目顺序**：必须严格按照知识点清单顺序出题，第N题必须对应清单[N]，知识点ID必须完全一致（违反此条会导致批改错误，复习进度全乱）
2. **不得跳步**：每个流程的步骤必须按顺序执行，特别是"运行脚本"和"自检"环节
3. **不得省略下划线**：填空题空白处必须有`____（N）____`，不能只写（1）不写下划线
4. **不得忘记运行脚本**：出题前必须跑quiz_pull.py，批改后必须跑quiz_push.py
5. **不得编造知识点**：所有题目必须来自today_quiz.json中的知识点核心内容，不得超纲
6. **不得修改grading.json格式**：必须严格按照模板格式，否则quiz_push.py会解析失败
7. **★☆☆规则**：补充级知识点只出选择题，不主动出题
8. **FIX保护规则**：批改时必须遵守FIX#1~#6，特别是"随机巩固答对不推进轮次""未到期巩固不计入连续答对清除"

## 九、Skill修改协议（⚠️修改本Skill时必须遵守）

> 当用户要求修改本Skill的任何规则、配置、格式、逻辑、文档，**或新增功能/内容**时，必须严格遵守以下协议，防止修改遗漏导致不一致。
>
> **核心原则**：修改任何内容时，不仅要修改业务内容本身，**修改规则相关的文档/脚本（即三位一体防护体系本身）也要同步更新**；新增内容必须参照本协议纳入防护体系。

---

### 修改前（必须执行）

1. **定位所有关联位置**：运行以下命令搜索要修改的关键词，找出所有需要同步修改的位置：
   ```powershell
   python d:\AI\AI学习内容记忆测试\ai-quiz-system\check_consistency.py where 关键词
   ```
   例如：要改"薄弱点到期"标签名，运行 `python check_consistency.py where 薄弱点到期`
2. **查阅映射表**：读取 `d:\AI\AI学习内容记忆测试\ai-quiz-system\content-map.md`，确认"主定义位置"（只改那里）和所有"引用位置"
3. **检查SSOT**：如果是常量/阈值/标签/字段名/格式串，主定义位置应该在 `scripts/config.py` 中
4. **⚠️自指检查（关键！）**：判断本次修改是否涉及"修改规则本身"，如果是，需要同步更新三位一体文件：
   - 如果修改了**校验规则** → 同步更新 `check_consistency.py` 中对应的检查项
   - 如果修改了**常量定义/新增了常量** → 同步更新 `content-map.md` 第一章节的常量表
   - 如果修改了**修改流程/本协议本身** → 同步更新 `content-map.md` 末尾的"修改流程Checklist"
   - 如果新增了**文件/目录/模板/清单** → 同步更新 `content-map.md` 第八章"文件清单"

### 修改中（必须遵守）

#### 普通业务内容修改
5. **常量只改config.py**：所有可配置常量（路径、标签、阈值、字段名、状态值、格式串、图标）只在 `scripts/config.py` 中定义，不要在quiz_pull.py/quiz_push.py中硬编码
6. **代码引用config**：脚本中必须使用 `from config import ...` 导入常量，不要直接写中文字符串或数字
7. **文档同步更新**：修改规则后必须同步更新 `references/` 下对应的规则文档
8. **清单文件同步**：如果修改了出题/批改规则，必须同步更新 `assets/checklists/` 下对应的自检清单

### 三位一体防护体系说明

| 防护层 | 文件 | 作用 | 自指更新方式 |
|--------|------|------|-------------|
| 第一层（源头） | `scripts/config.py` | 所有常量唯一来源，改一处全生效 | 新增常量需同步更新content-map.md常量表+check_consistency.py的required_constants |
| 第二层（地图） | `content-map.md` | 人工查阅的关联地图，标注每个内容的主定义和引用位置 | 修改映射表后需同步更新SKILL.md路径表和check_consistency.py的检查逻辑 |
| 第三层（兜底） | `check_consistency.py` | 自动校验55+项规则，拦截遗漏和不一致 | 新增检查项需同步更新content-map.md对应章节的"修改注意事项" |

**术语解释**：
- **自指更新**：当修改的内容涉及到"修改规则本身"（即config.py/content-map.md/check_consistency.py/SKILL.md这四个三位一体核心文件）时，不仅要改业务内容，还要同步更新防护体系本身，确保护栏始终有效
- **SSOT（Single Source of Truth）**：单一事实来源，即所有常量只在config.py中定义一次，其他地方通过import引用

**违反本协议的后果**：修改一处漏改多处，导致脚本逻辑与文档规则不一致、格式错误、飞书写回失败、复习间隔计算错误；更严重的是，修改防护体系本身后不更新，会导致后续修改失去保护，问题越积越多。
